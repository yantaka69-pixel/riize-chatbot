# RIIZE 虚拟聊天模拟器 - 产品需求文档

## 1. 产品概述

一个专为 RIIZE 粉丝打造的沉浸式虚拟聊天平台，用户可以选择 RIIZE 不同成员，体验类似 Bubble/Weverse 私信的虚拟聊天互动。

**核心价值：** 突破次元壁，让粉丝与虚拟化的 RIIZE 成员进行一对一深度互动。

## 2. 页面流程

```
打开网页 → 输入昵称 → 选择今日心情 → 成员选择页 → 选择成员 → 选择聊天模式 → 进入聊天
```

## 7. 亲密度系统

### 等级和称呼

| 等级 | 称呼 | 分数范围 |
|------|------|----------|
| Lv.1 | 初识粉丝 | 0-99 |
| Lv.2 | 眼熟粉丝 | 100-299 |
| Lv.3 | 常来聊天的人 | 300-599 |
| Lv.4 | 熟悉朋友 | 600-999 |
| Lv.5 | 亲近粉丝 | 1000-1999 |
| Lv.6 | 特别在意的人 | 2000-3999 |
| Lv.7 | 专属陪伴 | 4000+ |

### 亲密度增加规则

- 每发送一条消息：+1~3点
- 每天首次聊天：+5点
- 特殊互动（情绪安慰/韩语练习）：+3点
- 亲密度不下降

## 8. AI 回复规则

### 语言规则

- 默认中文为主
- 偶尔出现韩语词（如"괜찮아""고마워"）
- 韩语练习模式下增加韩语比例
- 不出现整段韩语

### 回复长度

- 用户短句：1-2句
- 用户倾诉多：3-6句
- 情绪安慰模式可稍长

## 9. 内容安全

### 禁止内容

- 冒充真实成员本人
- 提供私人联系方式
- 编造私人行程/恋爱关系/家庭隐私
- 诱导线下见面
- 输出成人内容

### 允许内容

- 粉丝向虚拟聊天
- 情绪陪伴
- 轻度恋爱感陪聊
- 虚拟日常表达

## 11. 数据库设计

### 数据表

- **users**: 用户表（id, nickname, device_id, today_mood）
- **members**: 成员配置表
- **conversations**: 聊天记录表
- **intimacy**: 亲密度表
- **settings**: 设置表

### 身份识别

- 用户 = 昵称 + 设备ID
- 设备ID自动生成并保存本地
- 同一设备可继续聊天记录
- 换设备视为新用户
